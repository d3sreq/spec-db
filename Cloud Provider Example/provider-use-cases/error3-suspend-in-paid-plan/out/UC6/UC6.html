UC6: Create account

  • User provides email, password, password confirmation and captcha.
  • System validates the provided data.
  • System sends email to given email address to verify that user has access to the given mailbox.
  • System notifies the user to proceed by following instructions in the sent email.
  • User opens the email and follows the link (the link points to the page that continues in user's registration).
  • System presents legal terms.
  • User accepts the terms.
  • System creates new user account with given email and password. #(emit:email) #emit:email
  • System assigns user free plan. #(mark:free_plan) #(emit:free_plan) #mark:mark_free_plan, #emit:free_plan
  • Use case finishes. #(mark:account_created) #mark:mark_account_created
Extension : 2a. At least one of the entries is missing or email in bad format or password too short or password confirmation doesn't match.
  • System shows message with given problem. #(abort) #abort
Extension : 2b. Email address already associated with account.
  • System shows message that the email address is already associated with an account. #(guard:account_created) #guard:mark_account_created
  • System sends email to the given email address with login instructions. #(abort) #abort
