UC9: Change plan

preceeded: UC7 Login
  • User initiates change of the plan.
  • System presents information about current plan and available plans.
  • User selects upgrade to paid plan. #(mark:wants_paid_plan) #mark:mark_wants_paid_plan
  • System evaluates that user has free plan associated and wants to upgrade to paid plan.
  • System checks that it has billing information of the user.
  • System checks that it has credit card of the user.
  • System presents review of the plan change and presents terms and conditions.
  • User confirms the plan change.
  • System computes first payment according to days left to the end of the current month and substracts it from the credit card.
  • System changed the plan to the paid one. #(unmark:free_plan) #(delete:free_plan) #unmark:mark_free_plan, #delete:free_plan
Variation : 3a. User wants to downgrade to free plan.
  • User selects downgrade to free plan. #(guard:!free_plan) #(unmark:wants_paid_plan) #guard:!mark_free_plan, #unmark:mark_wants_paid_plan
Variation : 4a. System evaluates that user has paid plan associated and wants to downgrade to free plan.
  • System skips billing information and credit card selection. #(guard:!free_plan & !wants_paid_plan) #(goto:7) #guard:(!mark_free_plan & !mark_wants_paid_plan), #goto:7
Variation : 4b. User selected same plan that has already associated.
  • System informs user that he has the selected plan already associated. #(guard:free_plan & !wants_paid_plan | !free_plan & wants_paid_plan) #guard:((mark_free_plan & !mark_wants_paid_plan) | (!mark_free_plan & mark_wants_paid_plan))
  • Use case aborts. #(abort) #abort
Extension : 5a. User wants to change billing information or the information has not been entered yet.
  • User provides billing information. #(include:UC12) #include:UC12
Extension : 6a. User wants to change credit card or the credit card has not been registered yet.
  • User provides credit card information. #(include:UC13) #include:UC13
Variation : 8a. User cancels the plan change.
  • Use case aborts. #(abort) #abort
Extension : 9a. Payment fails.
  • System shows message to user that the payment failed.
  • Use case aborts. #(abort) #abort
Variation : 10a. System changed the plan to free.
  • Use case finishes. #(guard:!wants_paid_plan) #(mark:free_plan) #(emit:free_plan) #guard:!mark_wants_paid_plan, #mark:mark_free_plan, #emit:free_plan
