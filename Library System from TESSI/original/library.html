Description of model library

1. Operators

The library can be operated by a user and a librarian.

2. Product Functions

The user can use the following functionality:

  • Reserve instance
  • borrow instance
  • Manage account
  • Extend rental period
  • return instance
  • Search medium

The librarian can use the following functionality:

  • Edit account
  • Manage medium
  • Create account
  • Delete account

2.1. Reserve instance

This function can be done by a user.

Procedure

  1. The user enters the shelfmark of the medium requested.
  2. The component (MediaAdministration)'s return list of the instances.
  3. The user chooses the instance.
  4. The user identifies to the system.
  5. A component (MediaAdministration) changes the status of the instance.

2.2. Edit account

This function can be done by a librarian.

2.3. borrow instance

This function can be done by a user.

Preconditions and Effects

If the MediumInstance is signed as available the user can borrow the instance. MediumInstance will afterwards be signed as borrowed.

Procedure

  1. The user identifies to the system.
  2. The user chooses the instance.
  3. A component (MediaAdministration) adds the instance to the borrowing list of the user account.
  4. A component (MediaAdministration) adds the user to the list of the users.
  5. A component (MediaAdministration) changes the status of the instance.
  6. A component (MediaAdministration) unlocks the instance.
  7. A component (MediaAdministration) receives the receipt for the instance newly borrowed.

2.4. Manage medium

This function can be done by a librarian.

2.5. Manage account

This function can be done by a user.

2.6. Create account

This function can be done by a librarian.

2.7. Delete account

This function can be done by a librarian.

2.8. Extend rental period

This function can be done by a user.

Preconditions and Effects

If the MediumInstance is signed as borrowed the user's Extend rental period can. MediumInstance will afterwards still be signed as borrowed.

Procedure

  1. The user identifies to the system.
  2. The component (MediaAdministration)'s return list of the instances.
  3. The user chooses the instance.
  4. A component (MediaAdministration) validates the new rental period.
  5. A component (MediaAdministration) extends the rental period and increases the total rental time.

2.9. return instance

This function can be done by a user.

Preconditions and Effects

If the MediumInstance is signed as borrowed the user can return the instance. MediumInstance will afterwards be signed as available.

Procedure

  1. The user chooses the instance.
  2. A component (MediaAdministration) locks the instance.
  3. A component (MediaAdministration) adds the return date to the list of the users.
  4. A component (MediaAdministration) removes the instance from the borrowing list of the user account.
  5. A component (MediaAdministration) changes the status of the instance.

2.10. Search medium

This function can be done by a user.

Procedure

  1. The user specifies the one search criteria.
  2. The component (MediaAdministration)'s search for the matching media.
  3. The component (MediaAdministration)'s return list of the instances.

3. System Behaviour

3.1. Instance states

A MediumInstance can be in one of the following states:

  • newly aquired
  • available
  • sorted out
  • borrowed
  • missing
  • borrowed and reserved
  • available and reserved

If the MediumInstance is signed as available the user can borrow the instance. MediumInstance will afterwards be signed as borrowed.

If the MediumInstance is signed as borrowed the user can return the instance. MediumInstance will afterwards be signed as available. Alternatively, the user's Extend rental period can. MediumInstance will afterwards still be signed as borrowed.
